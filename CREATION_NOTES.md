## 设计理念

### 核心洞察

在传统电商体验中，用户需要：打开浏览器 → 搜索商品 → 选择支付方式 → 输入卡号 → 确认支付。这个流程冗长且割裂，尤其对于加密货币用户来说，他们需要在钱包和电商平台之间反复切换。

**即付** 的核心理念是：**钱包本身就是最好的电商入口**。通过 AI 意图交互 + 自托管签名，用户可以在钱包内完成从发现商品到支付的全流程，无需离开钱包应用。

### 设计原则

1. **自然语言优先** — 用户不需要学习界面，只需要说出需求
2. **安全透明** — 每一步都让用户清楚知道发生了什么
3. **主权至上** — 用户始终拥有完全的资产控制权
4. **流畅体验** — 即使是演示项目，交互也要像真实产品一样流畅

## 技术实现

### Token Core (tcx-wasm) 集成

tcx-wasm 是 Token Core 的 WebAssembly 版本，提供了完整的密钥管理和交易签名能力。本项目的集成方式：

1. **钱包创建**: 使用 `create_keystore` 生成新的助记词和 keystore
2. **账户派生**: 使用 `derive_accounts` 从 keystore 派生以太坊地址
3. **交易签名**: 使用 `sign_tx` 在浏览器端完成交易签名
4. **助记词导出**: 使用 `export_mnemonic` 展示助记词（带安全提示）

所有操作均在浏览器端 WASM 环境中完成，密钥材料永远不会离开用户设备。

### Token UI Security 材料应用

深度参考了 Token UI Security SKILL.md 中的安全设计理念：

1. **核心威胁参考**:
   - 恶意合约交互与盲签 → 实现签名前完整交易解码展示
   - 钓鱼与地址欺骗 → 多层地址确认机制
   - 授权滥用 → 明确展示交易详情和风险

2. **签名前解码与展示**:
   - 完整展示收款地址、金额、网络信息
   - 防止用户在不知情的情况下签署恶意交易

3. **安全检查清单**:
   - 支付前强制完成 5 项安全检查
   - 每项检查都有详细说明和风险提示

### AI 意图解析

采用本地规则引擎实现意图解析，支持：
- 购买意图（"给我买一张..."）
- 搜索意图（"有没有..."）
- 浏览意图（"有什么推荐"）
- 话费充值意图（"帮我充值..."）

解析流程：关键词提取 → 商品匹配 → 金额识别 → 置信度评估 → 响应生成

## 项目亮点

1. **完整的交互闭环** — 从浏览到支付，每一步都可操作
2. **tcx-wasm 深度集成** — 真实的钱包创建、导入、签名流程
3. **多层安全设计** — 参考 Token UI Security 材料的安全检查
4. **AI 意图交互** — 展示下一代钱包的交互形态
5. **高质量 Mock 数据** — 20+ 真实商品数据，包含热门礼品卡
